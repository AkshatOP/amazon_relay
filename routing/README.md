# Amazon Relay — Routing Module (Phase 3)

Self-contained geo-routing package. Consumes the grading agent's JSON and decides what to do
with a returned item: **RESELL_LOCAL / REFURBISH / DONATE / LIQUIDATE**.

## One-shot run sequence (from repo root)

```bash
# 1. Activate the venv
source .venv/bin/activate

# 2. Seed the SQLite database (pending orders, NGOs, returns_log table)
python -m routing.db.seed_db

# 3. Generate 8000-row synthetic training data
python -m routing.model.generate_training_data

# 4. Train the XGBoost router  ← YOU run this step
python -m routing.model.train_router

# 5. Start the grading API (port 8000, unchanged)
uvicorn backend.main:app --reload

# 6. Start the routing API (port 8100)
uvicorn routing.route_api:app --reload --port 8100

# 7. Serve the routing demo UI (port 5500)
python -m http.server 5500 --directory frontend_routing
```

## Ports

| Port | Service                        |
|------|-------------------------------|
| 8000 | Grading API (`backend/`)       |
| 8100 | Routing API (`routing/`)       |
| 5500 | Routing demo UI (`frontend_routing/`) |

**Port 8000 is reserved.** The routing service never touches `backend/main.py`.

## Before training (rule-fallback mode)

Before step 4 (`train_router.py`) is run, `router_model.pkl` does not exist. The router
automatically falls back to the same rule engine used to label the training data
(`decided_by: "rule_fallback"`). The API works end-to-end in this mode — you can demo
routing before training.

## API endpoints (port 8100)

### `GET /health`
Returns service status + whether the XGBoost model is trained and the DB is ready.

### `POST /route`
Route a pre-graded item.

```json
{
  "grade_json": {
    "grade": "B", "score": 7, "confidence": 0.9,
    "defects": [], "resale_eligible": true,
    "refurbish_recommended": false,
    "reasoning": "minor scuff only", "notes": "clean", "path": "visual"
  },
  "order_meta": {
    "order_id": "ORD-001", "asin": "B01DEMO",
    "product_name": "Running Shoes", "category": "footwear",
    "original_price": 3500, "age_months": 4,
    "customer_lat": 12.9166, "customer_lng": 77.6101
  }
}
```

### `POST /grade-and-route`
Accepts `order_meta` + an optional pre-supplied `grade_json`. Since image bytes travel via
multipart to `:8000/grade`, this endpoint expects a grade you obtained separately. It pings
`:8000/health` and reports whether the grader is alive.


## Economics honesty note

In **metro Bengaluru** (FC ~22 km away), the middle-mile truck is so efficient per item (250
items shared) that local-intercept savings are modest (~₹5/item) and CO2 can be slightly
*negative* (two local legs emit a touch more than one shared long-haul leg). The real impact
story is **tier-3**, where the FC is ~612 km away — there, savings are ~₹41/item and ~1 kg
CO2/item. The model reports both honestly; see `economics.py`'s self-test.
